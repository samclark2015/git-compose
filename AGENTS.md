# AGENTS.md

## Language & Toolchain

- Go 1.26.1 (managed via `mise`). Module name: `git-compose`.
- CGO must be disabled for production builds: `CGO_ENABLED=0`.
- No task runner (no Makefile, Taskfile, justfile). Use `go` directly.

## Build Commands

```bash
# Local dev build (current OS/arch, version="dev", self-update disabled)
go build -o git-compose ./cmd/git-compose

# Production build (linux/arm64, matches what CI releases)
GOOS=linux GOARCH=arm64 CGO_ENABLED=0 \
  go build -trimpath \
  -ldflags="-s -w -X main.githubRepo=samclark2015/git-compose -X main.version=2026.06.1" \
  -o git-compose-linux-arm64 \
  ./cmd/git-compose
```

No linter beyond `go vet`. Run `gofmt -w .` before committing.

## Tests

**There are zero test files in this repo.** `go test ./...` produces only `[no test files]`. New code additions should include tests.

## Package Layout

```
cmd/git-compose/   # main package — all CLI commands and business logic
  main.go          # kong CLI definition; build-time `version` var
  reconcile.go     # runReconcile() orchestrator (top-level flow)
  caddy.go         # Caddy Admin API integration
  docker.go        # Docker SDK: network creation, image pruning
  git.go           # git fetch/reset, hook installation, changed-service detection
  update.go        # self-update via GitHub Releases API + syscall.Exec re-exec
  util.go          # run(), runOutput(), envOr(), copyFile() helpers
internal/ui/
  ui.go            # Section/Step/OK/Warn/Fail/Info — colored terminal output
                   # Warn() goes to stderr; all others to stdout
```

Single binary, no monorepo, no workspaces.

## Execution Flow

```
reconcile → selfUpdate → gitSync → installHooks → ensureNetwork
         → deployServices (per services/*/compose.yaml, skip *.disabled)
         → waitForCaddy → applyCaddyRoutes → pruneImages
```

`selfUpdate()` runs first and may `syscall.Exec` re-exec the process with the new binary — code after it in the same process does not run if an update occurs.

## Key Conventions

- Errors: `fmt.Errorf("context: %w", err)` for wrapping; `errors.Join()` for multi-service failures. Non-fatal paths call `ui.Warn()` and continue.
- No interfaces or dependency injection — all functions are package-level.
- `//nolint:noctx` comments on HTTP calls are intentional; don't remove them or add context propagation without understanding the self-update flow.

## Versioning & Releases

- CalVer: `YYYY.MM.N` (e.g., `2026.06.3`), generated by CI from existing tag count.
- CI (`.github/workflows/build.yml`) builds `linux/arm64` on every push to `main`, creates a GitHub Release, and attaches the binary. PRs build but do not release.
- Go version for CI comes from `go.mod` via `go-version-file`.

## Runtime Requirements (not this repo, but the managed homelab)

The binary assumes the environment where it runs has:
- `docker` and `docker compose` on `PATH`
- `git` on `PATH`
- Docker socket access
- SOPS key material via env (`SOPS_AGE_KEY`, `AWS_*`, etc.)

Key env vars the binary reads at runtime: `REPO_DIR`, `CADDY_API`, `CADDY_NET`, `CADDY_ADMIN_LISTEN`, `CADDY_HTTP_LISTEN`, `CADDY_SERVER_NAME`, `CADDY_POLL_ATTEMPTS`, `CADDY_POLL_INTERVAL`.
